title: "Bill Payments"
---

Facilitate automated bill payment services for utilities, subscriptions, and other recurring expenses. Use virtual accounts to collect payments from users and route funds to billers automatically. Support batch transfers to process multiple payments efficiently, reducing operational overhead while ensuring timely settlements.

**‚è±Ô∏è Time to complete:** 25-35 minutes

**üéØ What you'll build:** An automated bill payment system where users deposit USD into a virtual account, funds convert to USDC, then batch transfers pay multiple billers (utilities, subscriptions, etc.) in a single transaction.

## Use Case Overview

This recipe demonstrates an automated bill payment flow where:

- ‚úÖ Users deposit USD into dedicated virtual accounts
- ‚úÖ USD automatically converts to USDC stablecoin
- ‚úÖ Batch transfers distribute payments to multiple billers
- ‚úÖ Operational overhead is minimized through automation

This flow enables scalable bill payment services with reduced processing costs and timely settlements.

## What you'll build

Here's what we'll accomplish in this recipe:

1. **Generate API Keys** - Set up authentication for HIFI API
2. **Create User** - Onboard a bill payer and complete KYC for USD rail
3. **Add Virtual Account** - Set up account for collecting USD payments that convert to USDC
4. **Execute Batch Transfer** - Pay multiple billers in a single transaction

**Payment Flow:**

```
USD ‚Üí User Bank Account ‚Üí Virtual Account ‚Üí USDC ‚Üí Batch Transfer ‚Üí Multiple Billers
```

## Create User (Bill Payer)

Create a verified user account for the bill payer. Every user must accept HIFI's Terms of Service and complete KYC verification before they can access fiat rails and make payments.

<Note>
  **Complete Onboarding Guide:** For a comprehensive walkthrough of user
  onboarding and KYC verification, see our [User Guide](/docs/users).
</Note>

### User Creation Flow

Creating a user involves four steps:

1. **Generate Terms of Service Link** - Get acceptance link for your customer
2. **Create User** - Create the user account after TOS acceptance
3. **Update KYC Information** - Provide required personal details
4. **Submit KYC** - Submit for verification to unlock USD rail

<AccordionGroup>
  <Accordion title="Generate Terms of Service Link">

Before creating a user, they must accept HIFI's Terms of Service. Generate a unique acceptance link using the [Generate Terms of Service Link](https://docs.hifibridge.com/v2/reference/post_v2-tos-link) endpoint.

**Request:**

```bash
curl --request POST \
  --url https://sandbox.hifibridge.com/v2/tos-link \
  --header 'accept: application/json' \
  --header 'authorization: Bearer YOUR_API_KEY' \
  --header 'content-type: application/json' \
  --data '
{
  "idempotencyKey": "c9c91801-d3a2-4a1c-aa2e-fcb2c473198c"
}
'
```

**Request Fields:**

<ResponseField name="idempotencyKey" type="string" required>
  A unique UUID that becomes your `signedAgreementId` after the user accepts the
  terms. Generate this using any [UUID
  generator](https://www.uuidgenerator.net/).
</ResponseField>

**Response:**

```json
{
  "signedAgreementId": "c9c91801-d3a2-4a1c-aa2e-fcb2c473198c",

}
```

**Response Fields:**

  Service.
</ResponseField>

<ResponseField name="signedAgreementId" type="string">

2. **Save the `signedAgreementId`** - You'll need this to create the user in the next step
3. **Wait for acceptance** - The `signedAgreementId` becomes valid only after the user clicks "Accept"

<Info>
  **Real-time Notifications:** HIFI triggers the `TOS_LINK.UPDATE` webhook event

## Add Virtual Account

Now that the user has passed KYC, create a **Virtual Account** for collecting bill payments. This is a bank account number automatically created by HIFI that users can deposit USD into. When fiat arrives, it's automatically converted to USDC and sent to their wallet, ready for batch distribution to billers.

<Note>
  **Virtual Account for Bill Payments:** Each virtual account comes with unique
  ACH/wire/RTP details you can surface in your app. When deposits arrive, HIFI
  detects the funds, applies your conversion settings (e.g., USD ‚Üí USDC), and
  makes them available for scheduled or batch payouts to billers.
</Note>

<AccordionGroup>
  <Accordion title="Create Virtual Account">

## Execute Batch Transfer

With deposits flowing into the virtual account and converting to USDC, create a **batch transfer** to pay multiple billers at once. Send one request containing every destination wallet address and amount; HIFI processes the entire batch and returns itemized statuses for your records.

<Note>
  **Batch Transfer Benefits:** Batch transfers reduce operational overhead by
  consolidating multiple payments into a single transaction. This is ideal for
  bill payment services where you need to distribute funds to multiple
  utilities, subscription services, or other billers on a regular schedule.
</Note>

<AccordionGroup>
  <Accordion title="Create Batch Transfer">

Create a batch transfer to pay multiple billers in a single transaction using the [Create Batch Transfer](https://docs.hifibridge.com/api-reference/crypto-transfer/create-batch-transfer) endpoint.

In this example, we'll pay three different billers (utilities, subscriptions, etc.) with 10 USDC each from the user's wallet.

**Request:**

```bash

  --url https://sandbox.hifibridge.com/v2/wallets/transfers/batches \
  --header 'accept: application/json' \
  --header 'authorization: Bearer YOUR_API_KEY' \
  --header 'content-type: application/json' \
  --data '
{
  "requestId": "b91772a6-f9c0-4479-a34c-c8723608d2e8",
  "currency": "usdc",
  "chain": "POLYGON",
  "source": {
    "userId": "3b77cec8-1bb9-5ea0-91a1-e8c1411dd429"
  },
  "destination": {
    "batch": [
      {
        "walletAddress": "0xA4C951a7ca4138220a9D4611e7A058d4673e14c3",
        "amount": "10"
      },
      {
        "walletAddress": "0x0466DCD644Ae2Ee5d88CF0e1976c3ce237A0D760",
        "amount": "10"
      },
      {
        "walletAddress": "0x8356d234077d4b7c71E35c95B075ed4ba5FF7681",
        "amount": "10"
      }
    ]
  },
  "requireApproval": false
}
'
```
