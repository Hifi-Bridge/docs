## Supported Currencies

### Fiat Currencies

- **USD** (US Dollar)
- **EUR** (Euro)
- **GBP** (British Pound)
- **CAD** (Canadian Dollar)
- **AUD** (Australian Dollar)

### Stablecoins

- **USDC** (USD Coin)
- **USDT** (Tether USD)
- **DAI** (Dai Stablecoin)

## Compliance and Security

### KYC Requirements

- Identity verification
- Address confirmation
- Source of funds documentation
- Ongoing monitoring

### AML Checks

- Transaction screening
- Sanctions list verification
- Suspicious activity monitoring
- Regulatory reporting

### Security Measures

- Multi-signature wallets
- Cold storage for reserves
- Insurance coverage
- Regular security audits

## Error Handling

Common conversion errors and solutions:

| Error                | Description                     | Solution                         |
| -------------------- | ------------------------------- | -------------------------------- |
| `INSUFFICIENT_FUNDS` | Not enough fiat balance         | Verify deposit completion        |
| `RATE_EXPIRED`       | Conversion rate no longer valid | Request new rate                 |
| `COMPLIANCE_FAILED`  | KYC/AML check failed            | Complete additional verification |
| `ACCOUNT_SUSPENDED`  | Account temporarily disabled    | Contact support                  |

## Best Practices

1. **Pre-verify Users**: Complete KYC before enabling conversions
2. **Monitor Limits**: Set appropriate daily/monthly conversion limits
3. **Rate Management**: Use rate locking for large transactions
4. **Error Handling**: Implement comprehensive error handling and user notifications
5. **Compliance**: Maintain detailed audit trails for all conversions
