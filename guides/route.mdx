# Route: Move Stablecoins Between Wallets and Chains

The Route workflow enables seamless movement of stablecoins across different wallets, chains, and protocols. This is essential for multi-chain operations, cross-chain transactions, and efficient capital management.

## Overview

Routing stablecoins involves:

- **Cross-chain transfers**: Moving assets between different blockchains
- **Wallet management**: Transferring between different wallet addresses
- **Protocol bridging**: Moving between different DeFi protocols
- **Optimization**: Finding the most efficient routes and lowest costs

## Supported Networks

### Layer 1 Blockchains

- **Polygon** (MATIC) - Layer 2 scaling
- **Arbitrum** (ARB) - Optimistic rollup

### Layer 2 Solutions

- **Arbitrum One** - Optimistic rollup
- **Optimism** - Optimistic rollup

## Step-by-Step Process

### 1. Check Route Availability

Before routing, verify available paths and costs:

```bash
# Get available routes
GET /routes?from_chain=ethereum&to_chain=polygon&token=USDC&amount=1000

# Response
{
  "routes": [
    {
      "id": "route_1",
      "from_chain": "ethereum",
      "to_chain": "polygon",
      "token": "USDC",
      "amount": 1000,
      "estimated_gas": "0.001",
      "bridge_fee": "2.50",
      "total_cost": "2.50",
      "estimated_time": "10-15 minutes"
    }
  ]
}
```

### 2. Create Route Transaction

Initiate the routing transaction:

```bash
# Create a route transaction
POST /routes
{
  "from_wallet": "0x1234...",
  "to_wallet": "0x5678...",
  "from_chain": "ethereum",
  "to_chain": "polygon",
  "token": "USDC",
  "amount": 1000,
  "route_id": "route_1"
}
```

### 3. Execute Bridge Operation

Process the cross-chain transfer:

```bash
# Execute bridge
POST /bridges
{
  "route_id": "route_1",
  "transaction_id": "tx_123",
  "user_signature": "0xabcd..."
}
```

**Bridge Process:**

1. Lock tokens on source chain
2. Generate proof of lock
3. Submit proof to destination chain
4. Mint equivalent tokens on destination
5. Confirm completion

### 4. Monitor Progress

Track the routing progress:

```bash
# Get route status
GET /routes/{route_id}/status

# Response
{
  "id": "route_1",
  "status": "in_progress",
  "current_step": "bridging",
  "progress": 75,
  "estimated_completion": "2024-01-15T10:45:00Z",
  "source_tx": "0x123...",
  "destination_tx": "0x456..."
}
```

## Routing Strategies

### 1. Direct Bridge

- **Use case**: Simple chain-to-chain transfers
- **Pros**: Fast, direct, lower fees
- **Cons**: Limited to supported bridge pairs

### 2. Multi-hop Routing

- **Use case**: Complex routing through multiple chains
- **Pros**: More options, potentially lower costs
- **Cons**: Higher complexity, longer time

### 3. Aggregated Routing

- **Use case**: Finding best rates across multiple bridges
- **Pros**: Optimal pricing, multiple options
- **Cons**: Requires more computation

## Security Considerations

### Bridge Security

- **Multi-signature validation**: Multiple signatures required
- **Time locks**: Delays for large transactions
- **Circuit breakers**: Automatic pause mechanisms
- **Insurance coverage**: Protection against bridge failures

### Transaction Security

- **Address validation**: Verify destination addresses
- **Amount limits**: Set maximum transfer amounts
- **Rate limiting**: Prevent rapid successive transfers
- **Monitoring**: Real-time transaction monitoring

## Error Handling

Common routing errors and solutions:

| Error                | Description                    | Solution                      |
| -------------------- | ------------------------------ | ----------------------------- |
| `INSUFFICIENT_GAS`   | Not enough gas for transaction | Increase gas limit or price   |
| `BRIDGE_UNAVAILABLE` | Bridge temporarily down        | Wait or use alternative route |
| `INVALID_ADDRESS`    | Destination address invalid    | Verify address format         |
| `AMOUNT_TOO_LARGE`   | Exceeds maximum transfer       | Split into smaller amounts    |
| `NETWORK_CONGESTED`  | High network congestion        | Wait or increase gas price    |

## Best Practices

1. **Route Planning**: Always check multiple routes for best pricing
2. **Gas Management**: Monitor gas prices and optimize timing
3. **Address Verification**: Double-check destination addresses
4. **Amount Limits**: Set appropriate limits for different use cases
5. **Monitoring**: Track all routing transactions
6. **Fallback Plans**: Have alternative routes ready
